# 테스트 우선순위 매트릭스

## 우선순위 기준

| 복잡도 | 도메인 중요도 | 우선순위 | 예시 |
|--------|--------------|---------|------|
| 높음 | 높음 | ⭐⭐⭐ 최우선 | 광고 매출 집계, 정산 로직 |
| 낮음 | 높음 | ⭐⭐ 중간 | 콘텐츠 등록 검증 |
| 높음 | 낮음 | ⭐ 낮음 | 유틸리티 함수 |
| 낮음 | 낮음 | - 생략 | Getter/Setter |

## 테스트 순서

```
1순위: 핵심 성공 케이스 (기능 동작 증거)
2순위: 주요 변형 케이스 (정액 → 정률)
3순위: 경계값 (최소금액 경계, 0원, 음수)
4순위: 예외 케이스 (만료, 중복사용)
5순위: 엣지 케이스 (동시성, 특수문자)
```

## 핵심 경로(Happy Path)부터 시작

```typescript
describe('CouponService', () => {
  describe('apply', () => {
    test('정액 쿠폰 적용 시 해당 금액만큼 할인', () => {
      // 가장 단순한 성공 케이스부터
    });
  });
});
```

## 경계값 체크리스트

```
숫자: 0, 1, -1, 최대값, 최소값, 경계 ±1
문자열: 빈 문자열, null, 최대 길이
날짜: 오늘, 어제, 내일, 경계 시점
컬렉션: 빈 배열, 1개, 최대 개수
```

## 2단계 체크로 필요성 판단

**1단계 (필수): 우리 코드인가?**
- ❌ → **테스트 불필요** (외부 라이브러리, 프레임워크 동작)
- ✅ → 2단계로

**2단계 (가치): 다음 중 하나라도 해당하는가?**
- [ ] **비즈니스 로직** (조건문, 계산, 변환, 집계)
- [ ] **복잡도** (경계값, 엣지케이스, 여러 단계 처리)

→ **하나라도 ✅ → 테스트 필요**
→ **모두 ❌ → 테스트 불필요** (단순 getter, passthrough)
