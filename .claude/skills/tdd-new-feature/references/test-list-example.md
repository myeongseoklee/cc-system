# 테스트 목록 예제

## 쿠폰 적용 기능

### 요구사항
- 주문에 쿠폰을 적용할 수 있다
- 쿠폰은 정액/정률 두 종류가 있다
- 최소 주문금액 미달 시 적용 불가
- 쿠폰은 1회만 사용 가능
- 만료된 쿠폰은 사용 불가

### 테스트 목록
```
□ 정액 쿠폰 적용 시 해당 금액만큼 할인
□ 정률 쿠폰 적용 시 비율만큼 할인
□ 최소 주문금액 미달 시 에러
□ 이미 사용된 쿠폰 적용 시 에러
□ 만료된 쿠폰 적용 시 에러
□ 할인 금액이 주문 금액 초과 시 0원 처리
□ 쿠폰 적용 성공 시 사용 처리
```

## 월별 회계 리포트 기능

### 요구사항
- 일별 매출 데이터를 월별로 집계
- 앱별로 충전코인 데이터 조회
- 집계 데이터를 Excel로 출력

### ❌ 테스트 목록에서 제외
```
❌ Repository에서 충전코인 조회
  → 우리 코드 ✅, 비즈니스 로직 ❌, 복잡도 ❌ → 제외

❌ ExcelJS가 Buffer 반환
  → 우리 코드 ❌ → 제외

❌ UseCase가 Repository 올바른 파라미터로 호출
  → 구현 세부사항 (Stub 호출 검증)
```

### ✅ 테스트 목록에 포함
```
✅ 일별 데이터 → 월별 데이터 집계
  → 우리 코드 ✅, 비즈니스 로직 ✅ (계산) → 포함

✅ 일별 데이터 없고 충전만 있는 앱 → 빈 회계 데이터 생성
  → 우리 코드 ✅, 비즈니스 로직 ✅ (조건부 로직) → 포함

✅ 회계 데이터를 Excel 행 형식으로 변환
  → 우리 코드 ✅, 비즈니스 로직 ✅ (변환) → 포함

✅ 날짜 경계값: 연도 전환 (1월 → 이전 해 12월)
  → 우리 코드 ✅, 복잡도 ✅ (엣지케이스) → 포함
```

## 핵심 원칙

**관찰 가능한 동작**만 테스트 목록에 포함

- ✅ **입력 → 출력 변환**: 계산, 변환, 집계, 필터링
- ✅ **조건부 분기**: if/else, switch, 예외 처리
- ✅ **경계값**: 0, null, 빈 배열, 날짜 경계
- ❌ **어떻게 구현하는가**: Repository 호출, 라이브러리 사용
- ❌ **중간 과정**: private 메서드, 내부 상태 변화
